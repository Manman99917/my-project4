# 代码规范

## 1. 代码风格
代码风格严谨，注重模块化、可读性和边界情况的处理。

## 2. 代码目录结构

目录结构：​ 请严格遵循以下结构，不要将所有代码写在 page中。

utils/：存放工具函数（如时间格式化 formatTime.js、通用的云数据库操作封装 dbService.js）。

config/：存放常量配置（如数据库集合名、错误码）。

pages/：存放页面。

数据层分离 (Service Layer)：​ 禁止在 Page 的 JS 文件中直接调用 wx.cloud.database()。必须封装一个 RecordService类或模块，专门负责数据的增删改查。Page 层只负责调用 Service 和更新 UI。

常量提取：​ 将 'poop'和 'shampoo'这样的字符串定义为常量（如 const RECORD_TYPE = { POOP: 'poop', SHAMPOO: 'shampoo' }），禁止使用魔法字符串。

错误处理：​ 所有的云函数调用和数据库操作必须有 try...catch包裹，并在失败时通过 wx.showToast给用户友好提示，而不是直接报错。

注释规范：​ 使用 JSDoc 格式编写函数注释，特别是复杂的计算逻辑（如时间间隔计算）。

功能实现细节：

一键记录：​ 点击按钮调用 RecordService.add(recordData)，记录成功后进行震动反馈（wx.vibrateShort）并刷新页面数据。

数据展示：

显示最近一次记录的具体时间（格式：YYYY-MM-DD HH:mm）。

显示距离上次的间隔时间（使用相对时间，如“2小时前”或“3天前”）。

如果无记录，显示“暂无记录”，间隔显示“未开始”。

云函数设计：​ 请编写一个云函数 getLatestRecord，接收参数 { type: 'poop' }，返回该类型的最新一条记录，而不是在前端拉取列表再排序，以减少数据传输量。
## 3. 输出格式要求
请按以下顺序输出代码：

utils/dbService.js(数据库操作封装)

config/constants.js(常量定义)

pages/index/index.js(首页逻辑，需引入上述模块)

pages/index/index.wxml(首页结构)

cloudfunctions/getLatestRecord/index.js(云函数逻辑)
